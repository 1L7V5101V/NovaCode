来自 `session_events.py:1-5` 的模块文档：

```
The run trace is per-task and diagnostic.
The session event bus is the durable, coarse-grained timeline for the interactive session itself.
```

区别：

|         | SessionEventBus (events.jsonl)           | Run Trace (trace.jsonl)               |
| ------- | ---------------------------------------- | ------------------------------------- |
| **范围**  | 整个会话生命周期                                 | 单次 `run_turn()` 执行                    |
| **粒度**  | 粗粒度（turn start/finish、worker 通知、todo 变更） | 细粒度（prompt 构建、model 请求/解析、tool 执行每一步） |
| **存储**  | `{session_id}.events.jsonl`              | `{run_dir}/trace.jsonl`（每次 run 独立目录）  |
| **目的**  | 会话 timeline，供 UI/TUI 消费                  | 审计/调试/报告，详细记录每次 LLM 交互和工具结果           |
| **持久性** | 整个 session 期间持续追加                        | 每次 run 独立，run 结束即完整                   |

一句话：**events 是会话的大事记，trace 是一次运行的流水账。**
