根据项目文档，10 个恢复场景按 **5 类边界 × 每类 2 个场景** 组织：

| 类别 | 场景 |
|------|------|
| **1. 基础 checkpoint 恢复** | ① 目标（goal）能从 checkpoint 带回<br>② 关键文件上下文能从 checkpoint 带回 |
| **2. 部分状态过期** | ③ 单文件锚点过期<br>④ 多文件锚点过期 |
| **3. workspace 漂移** | ⑤ workspace 指纹不匹配（文件层不一致）<br>⑥ runtime 身份不匹配（cwd、model 等变化） |
| **4. checkpoint 自身不兼容** | ⑦ schema 版本不兼容（结构变化）<br>⑧ 完全无可用 checkpoint（唯一故意不恢复的场景） |
| **5. 工具半成功恢复** | ⑨ shell 半成功后恢复<br>⑩ 工具半成功后恢复 |

详情见 `后端agent开发资料\归档\【重要】90-针对项目描述的逐字面试话术.md:479-501` 和 `pico/metrics.py:1287-1348`。
