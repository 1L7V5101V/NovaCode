不是。分三种情况：

---

### 1. `prefix` / `memory` / `skills` — 直接截断末尾

```python
# context_manager.py:240
rendered_text = tail_clip(raw, int(budget)) if budget is not None else raw
```

`tail_clip()` 的实现（`turn_history.py:7-15`）：

```python
def tail_clip(text, limit):
    if len(text) <= limit:
        return text
    if limit <= 3:
        return text[:limit]
    return text[: limit - 3] + "..."
```

**是的，就是直接裁掉末尾。**

---

### 2. `relevant_memory` — 按笔记均分预算

```python
def _render_relevant_memory(self, selected_notes, budget):
    per_note_budget = self._per_note_budget(budget, len(note_texts), header)
    rendered_notes = [tail_clip(text, per_note_budget) for text in note_texts]
```

先把预算平分给每条笔记，然后**每条笔记各自从末尾 tail_clip**。Note 之间是平等的，不会被一条长笔记挤占所有空间。

---

### 3. `history` — 智能选择

```python
# turn_history.py:45-78

            continue
        if entry["turn_id"] in recent_turns:
            clipped = [tail_clip(line, ...) for line in entry["lines"]]
            # 最近轮次尝试逐行 clip 后保留
        # 旧轮放不下 → 直接丢弃这一整轮
```

判定过程：

```
从最旧的轮次开始逆序尝试加入渲染列表

                    ┌─────────────────────────┐
                    │  加入整轮 → 不超 budget?  │
                    ├──────┬──────────┘
                    Yes   No
                     │     │
                     ▼     ▼
                  保留整轮  最近 3 轮? ──Yes──→ 每行 tail_clip 后尝试
                            │               
                            No
                            │
                            ▼
                         直接丢弃整轮
```

所以 history section 的裁剪策略是 **"选择性的轮次丢弃 + 最近轮次行级 clip"**，不是简单从末尾截断。

---

总结：

| Section | 裁剪方式 |
|---|---|
| `prefix` | 直接裁末尾 |
| `memory` | 直接裁末尾 |
| `skills` | 直接裁末尾 |
| `relevant_memory` | 按笔记均匀分配预算，各自裁末尾 |
| `history` | 先尝试保留完整轮次，放不下则丢弃旧轮，最近轮再逐行 clip |
| `current_request` | **不裁剪** |
