当父 agent **没有** `model_client_factory` 时走同步阻塞（`worker_manager.py:47`）：

```python
run_worker(self, task, prompt, action="spawn")  # 直接在当前线程执行
```

这意味着 `tool_agent()` 不会返回，`execute_tool_payload()` 不会 yield，**整个父 agent 的 engine loop 停在那里等子 agent 跑完**。子 agent 完整跑完自己的 `run_turn()` 循环（build prompt → call model → execute tool → ... → final），`run_worker()` 才返回，父 agent 拿到结果，继续下一轮。

反之如果有 `model_client_factory`（`_can_run_background()` 返回 True），就走后台线程：`spawn()` 立刻返回 `{"status": "started"}`，子 agent 在独立线程里跑，跑完结果塞进 notification queue，父 agent 下一轮 drain 时拿到通知。
